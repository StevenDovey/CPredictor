# CSV Input Templates for CPredictor

These CSV files replace the Excel input workbook sheets. You can use either
CSV files **or** the original `.xlsx` workbook — the R code auto-detects the
format from the file extension.

## The 3 user input sheets

In the multi-species Excel model, the user fills in 3 sheets:

1. **Inputs / Growth model** — Stand info (species, stocking, rotation length),
   thinning schedule (up to 4 thins), pruning schedule (up to 4 lifts),
   metrics for estimating site productivity indices (age, stocking, height,
   BA/DBH), and environmental parameters (latitude, elevation, soil, temperature).

2. **Starting tree list** — Plot area (ha), measurement age, and individual
   tree DBH (cm) and height (m). Heights can be blank.

3. **C Change** — Regime (species, stocking, rotation, thinning, pruning),
   calibration measurement (age, stocking, BA, MTH), site info (latitude,
   elevation, soil C/N/P, temperature), wood density options, harvest removal
   percentages, and NuBalM toggle.

The **300 Index** sheet is **model-generated output** — it is populated by the
calibration and growth model, not filled in by the user.

## Multi-sheet workbook → directory of CSVs

The Excel workbook `input.xlsx` contains multiple sheets. To use CSVs instead,
create a **directory** (e.g. `my_inputs/`) and place one CSV per sheet:

| Excel sheet          | CSV filename              | User input? | Description                                    |
|----------------------|---------------------------|-------------|------------------------------------------------|
| `Inputs`             | `inputs.csv`              | Yes         | Stand info, thinning/pruning, site parameters  |
| `Starting tree list` | `starting_tree_list.csv`  | Yes         | Individual tree DBH and height measurements    |
| `C Change`           | `c_change.csv`            | Yes         | Regime, calibration, site, carbon options      |
| `parameters`         | `parameters.csv`          | No          | Model coefficients (usually not user-edited)   |
| `VolTab`             | `voltab.csv`              | No          | Volume table coefficients (usually not edited)  |
| `300 Index`          | (not needed)              | No          | Model output — generated by calibration        |

Then pass the directory path instead of the `.xlsx` path:

```r
# Instead of:
run_full_chain(input_workbook = "input.xlsx")

# Use:
run_full_chain(input_workbook = "my_inputs/")
```

## Single-file CSVs

For pipeline steps that read a single-sheet file, just use `.csv` directly:

```r
# Tree data
run_tree_to_plot_summary(tree_file = "tree_data.csv")

# Plot site data
run_tree_to_plot_summary(plot_site_file = "plot_site_data.csv")

# Growth / yield / carbon intermediate files
run_yield_from_growth(growth_file = "growth.csv")
run_carbon_from_yield(yield_file = "yield.csv")
```

## Spatial raster files

Raster inputs (DEM, soil C/N/P, temperature) remain as GeoTIFF `.tif` files.
These are **not** converted to CSV.

## File descriptions

### inputs.csv
Two-column format: `field,value`. Contains stand information (species,
stocking, rotation length), thinning schedule (up to 4 thins), pruning
schedule (up to 4 lifts), site productivity options, measurement metrics
for estimating indices, and environmental parameters (latitude, elevation,
soil, temperature).

### starting_tree_list.csv
Row 1: `Plot_area_ha,<value>`
Row 2: `Age_years,<value>`
Row 3: blank
Row 4+: `Stem_number,DBH_cm,Height_m` (header + data rows).
Height can be blank for stems without height measurements.

### c_change.csv
Two-column format: `field,value`. Contains regime information (species,
stocking, rotation lengths, thinning, pruning), calibration measurement
(age, stocking, BA, MTH), site data (latitude, elevation, soil C/N/P,
temperature), wood density options, harvest removal percentages, initial
litter stocks, and NuBalM toggle.

### plot_site_data.csv
Standard tabular CSV with columns:
`Plot_Id, Latitude, Longitude, Easting, Northing, Elevation, Soil_C, Soil_N, Soil_P, Mean_Temp, Species`
