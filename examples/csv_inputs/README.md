# CSV Input Templates for CPredictor

These CSV files replace the Excel input workbook sheets. You can use either
CSV files **or** the original `.xlsx`/`.xlsm` workbook — the R code auto-detects
the format from the file extension.

## Two Excel models

There are two Excel models with different input structures:

### FCP_5_2.xlsm — Batch PSP model (recommended for batch runs)

The 3 user input sheets are:

1. **C_Change control** — Run settings: PSP Summary row range, rotation
   lengths, Y/N toggles (run C-Change, estimate drift factor, analyse
   only last measurement, run NuBalM).

2. **PSP Summary** — Per-plot measurement and regime records. Each row has
   a Type code: `E` (establishment/initial stocking), `M` (measurement),
   `TW` (waste thinning), `TP` (production thinning), `P1`–`P5` (pruning
   lifts). Columns: Plot, Type, Age, Stocking, BA, MTH, Pruned stems,
   Pruned height.

3. **Plots** — Site info per plot: Plot, Species, Year planted, Latitude,
   Elevation, Needle retention, Soil C/N/P, Early survival, Mean Temperature,
   Outerwood Density, density age, Inner/Outer ring, 300 Index, Site Index,
   Drift, Mortality coefficients, and initial litter stocks (DM/N/P for
   needle, branch, stem, coarse root, fine root).

### Multi-Species-Carbon-Calculator — Single-site model

The 3 user input sheets are:

1. **Inputs / Growth model** — Stand info (species, stocking, rotation length),
   thinning schedule (up to 4 thins), pruning schedule (up to 4 lifts),
   metrics for estimating site productivity indices, and environmental
   parameters (latitude, elevation, soil, temperature).

2. **Starting tree list** — Plot area (ha), measurement age, and individual
   tree DBH (cm) and height (m). Heights can be blank.

3. **C Change** — Regime (species, stocking, rotation, thinning, pruning),
   calibration measurement (age, stocking, BA, MTH), site info (latitude,
   elevation, soil C/N/P, temperature), wood density options, harvest removal
   percentages, and NuBalM toggle.

The **300 Index** sheet is **model-generated output** in both models — it is
populated by the calibration and growth model, not filled in by the user.

## Multi-sheet workbook → directory of CSVs

To use CSVs instead of an Excel workbook, create a **directory** and place one
CSV per sheet. The R code auto-detects directories vs files.

### For FCP_5_2 batch model (`run_batch_psp()`):

| Excel sheet          | CSV filename              | User input? | Description                                    |
|----------------------|---------------------------|-------------|------------------------------------------------|
| `C_Change control`   | `c_change_control.csv`    | Yes         | Run settings, row range, Y/N toggles          |
| `PSP Summary`        | `psp_summary.csv`         | Yes         | Per-plot measurements and regime records       |
| `Plots`              | `plots.csv`               | Yes         | Site info per plot                              |
| `parameters`         | `parameters.csv`          | No          | Model coefficients (usually not user-edited)   |
| `VolTab`             | `voltab.csv`              | No          | Volume table coefficients (usually not edited)  |

```r
# From Excel:
run_batch_psp(input_source = "FCP_5_2.xlsm")

# From CSV directory:
run_batch_psp(input_source = "my_csv_dir/")
```

### For multi-species single-site model (`run_full_chain()`):

| Excel sheet          | CSV filename              | User input? | Description                                    |
|----------------------|---------------------------|-------------|------------------------------------------------|
| `Inputs`             | `inputs.csv`              | Yes         | Stand info, thinning/pruning, site parameters  |
| `Starting tree list` | `starting_tree_list.csv`  | Yes         | Individual tree DBH and height measurements    |
| `C Change`           | `c_change.csv`            | Yes         | Regime, calibration, site, carbon options      |
| `parameters`         | `parameters.csv`          | No          | Model coefficients (usually not user-edited)   |
| `VolTab`             | `voltab.csv`              | No          | Volume table coefficients (usually not edited)  |
| `300 Index`          | (not needed)              | No          | Model output — generated by calibration        |

```r
# From Excel:
run_full_chain(input_workbook = "input.xlsx")

# From CSV directory:
run_full_chain(input_workbook = "my_inputs/")
```

## Single-file CSVs

For pipeline steps that read a single-sheet file, just use `.csv` directly:

```r
# Tree data
run_tree_to_plot_summary(tree_file = "tree_data.csv")

# Plot site data
run_tree_to_plot_summary(plot_site_file = "plot_site_data.csv")

# Growth / yield / carbon intermediate files
run_yield_from_growth(growth_file = "growth.csv")
run_carbon_from_yield(yield_file = "yield.csv")
```

## Spatial raster files

Raster inputs (DEM, soil C/N/P, temperature) remain as GeoTIFF `.tif` files.
These are **not** converted to CSV.

## File descriptions

### c_change_control.csv (FCP_5_2 batch model)
Two-column format: `field,value`. Row 3 = first PSP row, row 4 = last PSP row,
row 5 = 1st rotation length, row 6 = 2nd rotation length. Rows 8-12 are Y/N
toggles for C-Change, detailed output, drift factor, last-measurement-only,
and NuBalM.

### psp_summary.csv (FCP_5_2 batch model)
Standard tabular CSV with columns: `Plot, Type, Age (years), Stocking (stems/ha),
BA (m2/ha), MTH (m), Pruned stems (stems/ha), Pruned height (m)`.
Type codes: E=establishment, M=measurement, TW=waste thin, TP=production thin,
P1-P5=pruning lifts 1-5.

### plots.csv (FCP_5_2 batch model)
Standard tabular CSV with 20+ columns per plot: Plot, Species, Year planted,
Latitude, Elevation, Needle retention, Soil C/N/P, Early survival (%),
Mean Temperature, Outerwood Density, density age, Inner/Outer ring, 300 Index,
Site Index, Drift, Mortality coefficients. Optional columns 21-35 for initial
litter stocks (DM/N/P for 5 compartments).

### inputs.csv (multi-species model)
Two-column format: `field,value`. Contains stand information (species,
stocking, rotation length), thinning schedule (up to 4 thins), pruning
schedule (up to 4 lifts), site productivity options, measurement metrics
for estimating indices, and environmental parameters (latitude, elevation,
soil, temperature).

### starting_tree_list.csv (multi-species model)
Row 1: `Plot_area_ha,<value>`
Row 2: `Age_years,<value>`
Row 3: blank
Row 4+: `Stem_number,DBH_cm,Height_m` (header + data rows).
Height can be blank for stems without height measurements.

### c_change.csv (multi-species model)
Two-column format: `field,value`. Contains regime information (species,
stocking, rotation lengths, thinning, pruning), calibration measurement
(age, stocking, BA, MTH), site data (latitude, elevation, soil C/N/P,
temperature), wood density options, harvest removal percentages, initial
litter stocks, and NuBalM toggle.

### plot_site_data.csv (tree-to-plot-summary pipeline)
Standard tabular CSV with columns:
`Plot_Id, Latitude, Longitude, Easting, Northing, Elevation, Soil_C, Soil_N, Soil_P, Mean_Temp, Species`
