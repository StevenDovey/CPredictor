# CPredictor R Model — How to Run

## Prerequisites

```r
install.packages(c("readxl", "writexl"))
# Optional (for lat/elev from coordinates):
install.packages(c("sf", "terra"))
```

## Input Formats

All inputs can be **CSV** or **Excel (.xlsx/.xlsm)**. The code auto-detects
based on file extension. Spatial raster data (DEM, soil C/N/P, temperature)
stays as GeoTIFF `.tif` files.

### Two Excel models

There are two Excel models with different input structures:

- **FCP_5_2.xlsm** — Batch PSP model. User input sheets: `C_Change control`,
  `PSP Summary`, `Plots`. Use `run_batch_psp()`.
- **Multi-Species-Carbon-Calculator** — Single-site model. User input sheets:
  `Inputs`, `Starting tree list`, `C Change`. Use `run_full_chain()`.

### Multi-sheet workbooks → directory of CSVs

For FCP_5_2 batch model:

| Excel sheet          | CSV filename              | User input? | Description                                    |
|----------------------|---------------------------|-------------|------------------------------------------------|
| `C_Change control`   | `c_change_control.csv`    | Yes         | Run settings, row range, Y/N toggles          |
| `PSP Summary`        | `psp_summary.csv`         | Yes         | Per-plot measurements and regime records       |
| `Plots`              | `plots.csv`               | Yes         | Site info per plot                              |

For multi-species single-site model:

| Excel sheet          | CSV filename              | User input? | Description                                    |
|----------------------|---------------------------|-------------|------------------------------------------------|
| `Inputs`             | `inputs.csv`              | Yes         | Stand info, thinning/pruning, site parameters  |
| `Starting tree list` | `starting_tree_list.csv`  | Yes         | Individual tree DBH and height measurements    |
| `C Change`           | `c_change.csv`            | Yes         | Regime, calibration, site, carbon options      |
| `parameters`         | `parameters.csv`          | No          | Model coefficients (usually not user-edited)   |
| `VolTab`             | `voltab.csv`              | No          | Volume table coefficients (usually not edited)  |
| `300 Index`          | (not needed)              | No          | Model output — generated by calibration        |

Pass a directory path instead of `.xlsx`/`.xlsm`:

```r
run_batch_psp(input_source = "my_csv_dir/")    # FCP_5_2 batch
run_full_chain(input_workbook = "my_csv_dir/")  # multi-species single-site
```

### Single-file CSVs

For steps that read a single-sheet file, just use `.csv` directly:

```r
run_tree_to_plot_summary(tree_file = "tree_data.csv",
                         plot_site_file = "plot_site_data.csv")
```

See `examples/csv_inputs/` for example CSV templates and a README.

## Entry Points

There are **three ways** to run the model, depending on what data you have:

---

### 1. Tree-level data → Plot Summary

If you have **individual tree measurement data** (DBH, height per tree) and want to calculate plot-level starting values (MTH, BA, SI, 300 Index):

```r
setwd("/path/to/your/data")  # folder with your data files
source("Tree_To_PlotSummary.R")

# With Excel:
run_tree_to_plot_summary(
  tree_file = "your_tree_data.xlsx",
  input_file = "input.xlsx",
  plot_site_file = "Plot_site_data.xlsx",
  output_file = "Plot_Summary.xlsx"
)

# With CSV:
run_tree_to_plot_summary(
  tree_file = "tree_data.csv",
  input_file = "input.xlsx",           # or a directory of CSVs
  plot_site_file = "plot_site_data.csv",
  output_file = "Plot_Summary.xlsx"
)
```

**Outputs**: `Plot_Summary.xlsx` with 3 sheets (plot_description, measurement_info, plot_level) and `Plot_site_data_complete.xlsx`.

---

### 2. Plot summary → Full model chain (single site)

If you already have **plot-level growth data** (Age, MTH, BA, Stocking) plus regime and site info:

```r
setwd("/path/to/your/data")
source("Run_Model_Chain.R")

# With Excel:
run_full_chain(
  input_workbook = "your_input.xlsx",
  species = "Radiata pine",
  use_cchange = TRUE,
  soil_c = 5.57,
  soil_n = 0.296,
  soil_organic_p = 333,
  MATEMP = 12,
  output_dir = "output"
)

# With CSV directory:
run_full_chain(
  input_workbook = "my_csv_inputs/",   # directory containing inputs.csv, 300_index.csv, etc.
  species = "Radiata pine",
  use_cchange = TRUE,
  output_dir = "output"
)
```

**Outputs** in `output/`:
- `yield_from_growth.xlsx` — yield table (Age, Stocking, DBH, BA, Vol, WoodDensity)
- `carbon_output.xlsx` — carbon results from CChange
- `felled_stems.xlsx` — felled stems from thinning/harvest
- `harvest_logs.xlsx` — log-making results
- `carbon_from_yield.xlsx` — carbon summary
- `model_chain_outputs.xlsx` — combined report

---

### 3. Batch PSP run (FCP_5_2 workflow — recommended for multiple plots)

This is the main batch workflow from FCP_5_2.xlsm. It reads the 3 user input
sheets (C_Change control, PSP Summary, Plots) and processes all plots:

```r
setwd("/path/to/your/data")
source("Run_Model_Chain.R")

# From Excel workbook:
results <- run_batch_psp(
  input_source = "FCP_5_2.xlsm",
  output_dir = "batch_output",
  output_format = "csv"      # or "xlsx"
)

# From CSV directory:
results <- run_batch_psp(
  input_source = "my_csv_inputs/",   # contains c_change_control.csv, psp_summary.csv, plots.csv
  output_dir = "batch_output"
)

# results is a list with: plots_processed, yield_tables, c_change_predictions, c_change_output
print(results$plots_processed)  # shows Age, I300, SI, carbon, Species per plot
```

**Outputs** in `batch_output/` (matching FCP_5_2.xlsm output sheets):
- `plots_processed.csv` — One row per plot: Age, 300 Index, Site Index, Total/AGL/BGL/DWL/FL carbon, Species
- `yield_tables.csv` — Annual yield per plot: Stocking, MTH, Crown Lth, Volume, BA, DBH, Mean Height (b4/aft thin)
- `c_change_predictions.csv` — Summary carbon: Rot 1 & 2 Total/AGL/BGL/DWL/FL (tC/ha), shrub understorey (if C-Change enabled)
- `c_change_output.csv` — Detailed C_Change: dry-matter pools, litter, LAI, density (only when detailed output = Y)

### 4. Batch run (multiple separate workbooks)

If you have separate input workbooks per site (multi-species model):

```r
setwd("/path/to/your/data")
source("Run_Model_Chain.R")

# Each site can be an xlsx or a CSV directory
sites <- c("site1.xlsx", "site2_csvs/", "site3.xlsx")

results <- run_batch(
  workbook_paths = sites,
  species = "Radiata pine",
  output_root = "batch_output"
)

print(results)  # shows success/failure per site
```

Each site gets its own subfolder under `batch_output/` with all output files.

## The 3 User Input Sheets

### FCP_5_2.xlsm (batch PSP model)

1. **C_Change control** — Batch run settings: first/last row in PSP Summary,
   1st/2nd rotation length, Y/N toggles (run C-Change, detailed output,
   estimate drift factor, analyse only last measurement, run NuBalM).

2. **PSP Summary** — Per-plot measurement and regime records. Each row has a
   Type code: `E` (establishment/initial stocking), `M` (measurement),
   `TW` (waste thinning), `TP` (production thinning), `P1`–`P5` (pruning
   lifts 1–5). Columns: Plot, Type, Age (years), Stocking (stems/ha),
   BA (m2/ha), MTH (m), Pruned stems (stems/ha), Pruned height (m).

3. **Plots** — Site info per plot (20 columns): Plot, Species, Year planted,
   Latitude (decimal degrees), Elevation (m), Needle retention score,
   Soil %C, Soil %N, Soil Organic P (mg/kg), Early survival (%),
   Mean Temperature (C), Outerwood Density (kg/m3), density age, Inner ring,
   Outer ring, 300 Index, Site Index, 300 Index Drift, Additive Mortality
   Coef., Mult Mortality Coef. Optional columns 21–35 for initial litter
   stocks (DM/N/P for needle, branch, stem, coarse root, fine root).

When using CSVs, the 3 sheets become `c_change_control.csv`, `psp_summary.csv`,
and `plots.csv` in a directory.

### Multi-Species-Carbon-Calculator (single-site model)

1. **Inputs / Growth model** — Stand information (species, stocking, rotation
   length), thinning schedule (up to 4 thins), pruning schedule (up to 4 lifts),
   metrics for estimating site productivity indices (age, stocking, height,
   BA/DBH), and environmental parameters (mortality, latitude, elevation, soil,
   temperature, drift factor).

2. **Starting tree list** — Plot area (ha), measurement age (years), and a table
   of individual trees with stem number, DBH (cm), and height (m). Heights can
   be blank for stems without height measurements.

3. **C Change** — Regime (species, stocking, rotation lengths, thinning, pruning),
   calibration measurement (age, stocking, BA, MTH), site info (latitude,
   elevation, soil C/N/P, temperature), wood density options, harvest removal
   percentages, initial litter stocks, and NuBalM toggle.

The **300 Index** sheet is **model-generated output** in both models — it is
populated by the calibration and growth model (`siteIndex()` → `Calc300Index()`
→ `OutputGrowth()`), not filled in by the user.

## What each R file does

| File | Role |
|------|------|
| `io_utils.R` | CSV/Excel input dispatcher (read_data, read_sheet, write_output) |
| `Tree_To_PlotSummary.R` | Tree data → plot summary (SI, 300 Index) |
| `TreeLevel_Input.R` | Species parameter lookup + `run_model()` orchestrator |
| `MultiSpecies_Growth.R` | 41 growth/yield/taper/carbon functions for all species |
| `Growth_From_PlotSummary.R` | Plot summary → growth predictions |
| `300index2025V1.2.R` | Core 300 Index growth + wood density model |
| `DouglasFir_500Index.R` | Douglas-fir monthly growth model |
| `03_Yield_From_Growth.R` | Growth → yield table |
| `04_Carbon_From_Yield.R` | Yield → carbon (simple + CChange) |
| `CChange_model.R` | Full 16-state carbon compartment model |
| `05_Model_Report.R` | Combine outputs into final report |
| `input_validation.R` | Input range/format checks |
| `Run_Model_Chain.R` | Master orchestrator (`run_batch_psp`, `run_full_chain`, `run_batch`) |

---

## Validation

To compare R outputs against Excel, run the same site through both and compare columns age-by-age:
- SI, I300 (or I500 for Douglas-fir)
- MTH, DBH, BA, Vol, Stocking
- Wood density
- Carbon pools (AGL, BGL, DWL, FL, Total)
